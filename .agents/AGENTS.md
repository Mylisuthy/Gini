# Reglas Globales del Proyecto

- **Control de Versiones y Commits:** Está estrictamente prohibido realizar commits, hacer merges o enviar cambios al repositorio remoto (`git push`) de GitHub/GitLab sin recibir autorización expresa y previa del usuario. Todo archivo generado debe dejarse en el entorno local (Unstaged o Staged) hasta que el usuario dé luz verde para el commit/push.
